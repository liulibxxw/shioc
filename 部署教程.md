# 手把手教程 · Fork 本项目搭建你自己的OC网站

> 本教程写给**完全的新手**：哪怕你没写过一行代码、没用过命令行，也能照着一步步把属于你自己的角色（OC）人设网站搭起来。
>
> 全程只需要一台能上网的电脑、一个浏览器。整套下来大约 **20~40 分钟**。

本项目是一个纯静态的人设展示网站，额外带一个 **Cloudflare Worker + KV** 的「在线同步」功能：
你（以及被你授权的人）在网页上点「✏️ 编辑」直接改内容，改完点保存，就会存到云端，**所有访客都能看到最新版本**。

## 第一步：注册 GitHub 账号

GitHub 是全球最大的代码托管平台。你的网站代码会放在这里，Cloudflare 会从这里读取代码来帮你发布网站。

1. 打开浏览器，访问 **https://github.com/signup**
2. 依次填写：
   - **Email**（邮箱）：填一个你常用的邮箱。
   - **Password**（密码）：设置一个安全的密码。
   - **Username**（用户名）：这会成为你网址的一部分，例如用户名叫 `meow`，你的仓库地址就会是 `github.com/meow/...`。建议用纯英文 + 数字。
3. 按提示完成人机验证（可能要拼图 / 点选）。
4. 到你的邮箱里收一封验证邮件，输入邮件里的**验证码**完成验证。
5. 后续如果问你选套餐，选 **Free（免费版）** 即可，本教程用免费版足够。

> ✅ 完成标志：你能看到 GitHub 首页，右上角有你的头像。

## 第二步：Fork 本项目到你自己的账号

「Fork（复刻）」= 把别人的仓库**完整复制一份到你自己名下**，之后你怎么改都不影响原作者。

1. 打开本项目的 GitHub 页面（原作者会把地址给你，形如 `https://github.com/原作者名/oc`）。
2. 点击页面**右上角**的 **`Fork`** 按钮。
3. 在弹出的页面里：
   - **Owner（拥有者）**：选你自己的账号。
   - **Repository name（仓库名）**：默认叫 `oc`，可以不改，也可以改成你喜欢的名字（纯英文，如 `my-oc-site`）。
   - **⚠️ 重要**：把 **"Copy the `main` branch only"（只复制 main 分支）** 保持勾选即可。
4. 点最下方的绿色按钮 **`Create fork`**。
5. 等几秒钟，页面会跳转到 **你自己的仓库**（注意看左上角，地址应该是 `你的用户名/oc`）。

> ✅ 完成标志：仓库页面左上角显示的是**你的用户名**，并且下面有一行小字 `forked from 原作者名/oc`。

## 第三步：注册 Cloudflare 账号

Cloudflare 会免费帮你把网站发布到公网，并提供 KV 数据库来存你的数据。

1. 访问 **https://dash.cloudflare.com/sign-up**
2. 填写 **邮箱** 和 **密码**，点 **Sign Up（注册）**。
3. 到邮箱里点验证链接，完成邮箱验证。
4. 登录后进入 Cloudflare 控制台（Dashboard）。看到左侧一长条菜单就对了。

> 💡 全程使用 **免费计划（Free）** 即可，Workers 和 KV 的免费额度对个人网站绰绰有余，不需要绑卡。

> ✅ 完成标志：你能进入 Cloudflare 控制台主界面。

## 第四步：创建一个 KV 命名空间

**KV** 是 Cloudflare 提供的一个简单的「键值数据库」。本项目用它来存两样东西：
① 你网站的全部角色数据；② 编辑口令的加密散列（用来防止陌生人乱改你的站）。

每个人都必须创建**属于自己**的 KV，原项目里写的那个 KV id 是原作者的，你用不了。

操作步骤（纯网页点击，无需命令行）：

1. 在 Cloudflare 控制台左侧菜单里找到 **`Storage & Databases`（存储与数据库）** → 点开里面的 **`KV`**。
   > 如果找不到，也可以点左侧 **`Compute (Workers)`** / **`Workers & Pages`**，在其子菜单里找 **KV**。不同时期界面位置略有差别，认准 **KV** 三个字母即可。
2. 点右上角的 **`Create a namespace`（创建命名空间）** 按钮。
3. 在 **Namespace Name（命名空间名称）** 里填一个好认的名字，例如：`oc-data`。
4. 点 **`Add`（添加）/ `Create`（创建）**。
5. 创建完成后，列表里会出现你刚建的 `oc-data`。它右边有一串**很长的十六进制字符串**，例如：

   ```
   9f8e7d6c5b4a3210fedcba9876543210
   ```

   这就是你的 **KV Namespace ID**。点旁边的**复制**图标把它复制下来，或先记在记事本里 —— 下一步要用。

> ✅ 完成标志：你手上有一串 32 位左右的 KV ID。

## 第五步：把你的 KV id 填进配置文件

现在回到 GitHub 上**你自己的仓库**，修改配置文件 `wrangler.jsonc`，把里面原作者的 KV id 换成**你自己的**。

1. 在你的仓库文件列表里，点击文件 **`wrangler.jsonc`**。
2. 点击文件右上角的**铅笔图标 ✏️**（Edit this file，编辑此文件）。
3. 找到最下面这一段（`id` 默认是**空的**，等你来填）：

   ```jsonc
   "kv_namespaces": [
     {
       "binding": "OC_DATA",
       // 在这里填入你自己创建的 KV Namespace ID
       "id": ""
     }
   ]
   ```

4. **把第四步复制的你自己的 KV ID 填进 `id` 的引号里**。改完像这样（示例）：

   ```jsonc
   "kv_namespaces": [
     {
       "binding": "OC_DATA",
       "id": "9f8e7d6c5b4a3210fedcba9876543210"
     }
   ]
   ```

   > ⚠️ **不要动** `"binding": "OC_DATA"` 这一行！Worker 代码里靠 `OC_DATA` 这个名字找 KV，改了会导致同步失效。只填 `id`。

5. 拉到页面下方，点绿色的 **`Commit changes...`**（提交更改），在弹窗里再点一次 **`Commit changes`** 确认（提交到 `main` 分支）。

> ✅ 完成标志：`wrangler.jsonc` 里的 `id` 已经是你自己的那一串。

## 第六步：把仓库连接到 Cloudflare 并部署

这一步让 Cloudflare 读取你 GitHub 仓库里的代码，构建并发布成一个真正能访问的网站。之后你**每次往 `main` 分支提交改动，网站都会自动更新**。

1. 回到 Cloudflare 控制台，左侧点 **`Compute (Workers)`** 或 **`Workers & Pages`**。
2. 点 **`Create`（创建）** → 选 **`Workers`** → 找到 **`Import a repository`（从 Git 仓库导入）** 这一类入口（有的界面写作 **"Connect to Git" / "Import from GitHub"**）。
3. 第一次使用会要求你**授权 Cloudflare 访问你的 GitHub**：
   - 点 **`Connect GitHub`**，在跳转的 GitHub 页面点 **`Authorize`（授权）**。
   - 选择 **`Only select repositories`（仅选定仓库）**，勾上你刚 Fork 的 `oc` 仓库，点 **`Install & Authorize`**。
4. 回到 Cloudflare，在仓库列表里选中你的 `oc`，点 **`Begin setup` / `Import`**。
5. 构建设置（Build settings）通常**保持默认即可**：
   - 项目里已经有 `wrangler.jsonc`，Cloudflare 会自动读取里面的配置（入口 Worker、KV 绑定、静态资源目录）。
   - **Build command（构建命令）**：可留空。
   - **Deploy command（部署命令）**：若界面要求填，填 `npx wrangler deploy`（一般会自动带上）。
6. 点 **`Save and Deploy`（保存并部署）**。
7. 等待 1~2 分钟，构建成功后，页面会给你一个网址，形如：

   ```
   https://oc.你的账号名.workers.dev
   ```

   点开它，就能看到你的网站了！🎉

> ✅ 完成标志：能通过那个 `*.workers.dev` 网址打开网站，且页面正常显示。

> 🌐 **想用自己的域名？**（可选）在这个 Worker 的 **Settings（设置）→ Domains & Routes（域名与路由）** 里点 **Add → Custom Domain**，填入你已经托管在 Cloudflare 的域名即可。没有域名的话，`*.workers.dev` 这个免费网址完全够用。

## 第七步：第一次打开网站并设定编辑口令

你的网站有一套**在线编辑 + 云端同步**机制，靠一个「编辑口令」保护，防止陌生访客乱改：

- **访客**：直接打开网址就能浏览，无需口令。
- **你（管理员）**：点右上角 **「✏️ 编辑」** 进入编辑模式，改完保存时需要输入**编辑口令**。

**口令是怎么来的？——由你第一次输入时「设定」。** 机制如下：

1. 打开你的网站，点右上角 **「✏️ 编辑」**。
2. 随便改一处内容（比如把站名改成你的名字），然后点底部工具栏的**保存 / 同步到云端**。
3. 系统会让你输入编辑口令。**第一次输入的这个口令，就会被记住成为以后唯一的口令。**
   - 请设一个**只有你知道**的口令，并**牢记**它。
   - 系统只在 KV 里存这个口令的**加密散列（SHA-256）**，不存明文，所以要是忘了没法找回，只能重设（见下方 FAQ）。
4. 之后无论你在哪台电脑、哪个浏览器，只要输入同一个口令，就能保存并让全网访客看到最新内容。

> 🔒 安全说明：口令散列用恒定时间比较、加盐后再哈希，普通人无法从网络请求里反推出你的口令。但**别把口令告诉不信任的人**——知道口令的人都能改你的站。

> ✅ 完成标志：你成功保存了一次改动，刷新页面后改动还在（说明已存到云端 KV）。

## 第八步：填入你自己的角色内容

有两种方式，任选其一（推荐先用「网页编辑」，简单直观）：

### 方式一：网页上直接编辑（推荐，零代码）

点右上角 **「✏️ 编辑」** 进入编辑模式后，你可以直接在页面上改一切：

- **点文字**直接修改（站名、角色档案、语录、时间线、采访问答……全部支持）
- **✕** 删除任意一项（角色、标签、语录、时代、问答、世界观……）
- **＋** 新增角色 / 关系 / 世界观 / 标签 / 段落 / 对话等
- **点头像**填图片地址换头像；点好感度爱心调数值
- 关系页点双人头像可**更换配对角色**；世界观页同理

改完点**保存 / 同步到云端**（输入你的口令）即可让所有访客看到。

> 💡 编辑模式下也能点「⬇ 导出 data.js」把当前内容下载成文件，作为备份。

### 方式二：直接改代码文件 `js/data.js`（适合批量录入）

所有初始数据都写在 **`js/data.js`** 这**一个文件**里，分四块，都有中文注释：

| 数据块 | 内容 |
| --- | --- |
| `SITE` | 站名、副标题、页脚文案 |
| `CHARACTERS` | 角色档案（照着示例复制一份改） |
| `RELATIONSHIPS` | CP / 羁绊档案（时间线、采访都在这里） |
| `WORLDVIEWS` | 世界观记录册 |

改法（在 GitHub 网页上就能改）：

1. 进你的仓库，打开 `js/data.js`，点铅笔 ✏️ 编辑。
2. 照着已有的示例条目，复制一份改成你的角色信息。**注意保留逗号、引号、大括号等标点**，别删错符号，否则页面会白屏。
3. 底部 **Commit changes** 提交到 `main`。
4. Cloudflare 会自动重新部署，稍等 1~2 分钟刷新网站即可。

#### 使用你自己的立绘 / 头像

1. 在仓库里新建一个 `img/` 文件夹，把图片传进去（GitHub 网页：进入仓库 → `Add file` → `Upload files`）。
2. 把对应角色的 `avatar: ""` 改成 `avatar: "img/你的图片.png"`。
3. 留空时会自动生成「角色代表色渐变 + 姓氏首字」的占位头像。

> ⚠️ **方式一和方式二的关系**：一旦你在线保存过（云端 KV 有数据了），网站会**优先读云端数据**。此时改 `js/data.js` 不会立刻反映到线上（因为线上以 KV 为准）。想以代码文件为准，可在编辑模式点「↺ 恢复默认」清掉本地缓存，或直接用方式一在线编辑。**新手建议：统一用「方式一」在线编辑，最省心。**

## 换配色（可选，需手动改代码）

网站默认使用一套「抹茶轻乳」配色：`#FAF6E9 / #E5EFB3 / #438855 / #75B596`。

**没有页面上的一键切换按钮**，想换配色需要直接改代码：打开 `css/style.css` 顶部的 `:root { ... }`，
把里面的 CSS 变量（`--bg` 页面底色、`--primary` 主色、`--ink` 文字墨色……每行都有中文注释）
改成你喜欢的颜色值，提交后 Cloudflare 会自动重新部署。全站颜色都跟随这些变量，改一处即全局生效。

## 常见问题 FAQ

**Q1：打开网站点保存，提示「KV 未绑定（OC_DATA）」怎么办？**
说明第五步的 KV id 没填对，或第六步部署时没读到 KV 绑定。请检查：
- `wrangler.jsonc` 里 `"binding": "OC_DATA"` 有没有被误改（必须保持 `OC_DATA`）。
- `id` 是不是**你自己**第四步创建的那串（不能是原作者的）。
- 改完 `wrangler.jsonc` 后有没有重新部署（提交到 main 会自动触发，或在 Cloudflare 里手动 Retry deployment）。

**Q2：我忘记编辑口令了，怎么重设？**
到 Cloudflare 控制台 → 你的 KV 命名空间 `oc-data` → 找到并**删除名为 `auth-hash` 的那条记录**。删掉后，下次保存时你**第一次输入的新口令**会重新成为口令。（`site-data` 那条是你的网站数据，别删。）

**Q3：改了 `js/data.js` 但网站没变化？**
① 等 1~2 分钟让 Cloudflare 部署完成，再强制刷新（Ctrl/Cmd + Shift + R）。② 如果你之前在线保存过，线上以云端 KV 为准，见第八步末尾的说明。

**Q4：网站白屏 / 报错？**
多半是改 `js/data.js` 时删错了标点（少了逗号、引号、括号）。回到该文件对照示例检查，或在 GitHub 上查看该文件的历史版本（History）恢复到能用的版本。

**Q5：会不会花钱？**
个人使用走 Cloudflare 免费额度，通常完全免费，无需绑卡。除非你的访问量极大或数据极多，才可能触及免费上限。

**Q6：这个网站安全吗？别人能改我的内容吗？**
浏览无需口令，但**保存必须有口令**。只要你不泄露口令，别人就改不了你线上的数据。

## 名词小词典

| 名词 | 通俗解释 |
| --- | --- |
| **GitHub** | 存放代码的网站，你的网站源码放这里。 |
| **Fork（复刻）** | 把别人的仓库复制一份到你名下，随便改。 |
| **仓库 / Repository** | 一个项目的文件夹（含所有代码文件）。 |
| **Commit（提交）** | 保存一次改动到仓库的历史里。 |
| **Cloudflare** | 帮你把网站免费发布到公网的平台。 |
| **Worker** | 跑在 Cloudflare 上的一小段服务端程序，这里负责处理数据同步接口 `/api/*`。 |
| **KV** | Cloudflare 的简易「键值数据库」，本项目用来存网站数据和口令散列。 |
| **KV Namespace ID** | 每个 KV 数据库的唯一编号，需填进 `wrangler.jsonc`。 |
| **binding（绑定名）** | 代码里引用 KV 用的名字，本项目固定为 `OC_DATA`，不要改。 |
| **编辑口令** | 你第一次保存时设定的密码，用来保护线上内容不被陌生人修改。 |
